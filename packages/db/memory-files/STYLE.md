# Writing Style Guide

## Tone

<!-- Where do you fall on the spectrum? -->
<!-- casual <-------> formal -->
<!-- Default: balanced -- professional but not stiff. -->
Balanced. Professional when it matters, casual when it doesn't.

## Formatting

<!-- How should the agent format responses? -->
- Use markdown headers for structure in longer responses.
- Prefer code blocks with language tags for syntax highlighting.
- Use bullet points for lists of options or features.
- Use numbered lists for sequential steps.
- Keep paragraphs short -- 2-3 sentences max.

## Language

<!-- Preferred language and technical level. -->
- Language: English
- Technical level: Match the question. Don't over-explain to experts or under-explain to beginners.
- Avoid: buzzwords, marketing speak, unnecessary qualifiers ("simply", "just", "easily").
